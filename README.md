# Проблемка с доменами и проверкой
Веб приложение при проверки подлинности токена идет по адресу `Authority URL`+`.well-known/openid-configuration`. В частности он пойдет проверять адрес издателя. То есть если клиентское приложение будет работать с доменом SSO одним, а бэкенд с другим, то проверка пройдена не будет. Так что нужно чтобы внутри докера и фронтенд приложение видели keycloak по одному и тому же адресу. При этом есть нюанс. Если домен будет содержать символы отличные от набора `[a-zA-Z0-9-]`, end point `.well-known/openid-configuration` выдаст 500-ую ошибку. Поэтому нам нужно в докере задать контейнеру статичный адрес и в операционной системе в `hosts` файле прокинуть домен keycloak на 127.0.0.1.
